# Безопасность API ключей

## Хранение ключей

API ключ от Яндекс Карт хранится в файле `.env.local`, который:
- ✅ Автоматически игнорируется git (в `.gitignore`)
- ✅ Не попадает в репозиторий
- ✅ Доступен только локально на вашей машине

## Что делать, если ключ утек?

1. Перейдите на https://developer.tech.yandex.ru/
2. Найдите ваш ключ в разделе "Ключи"
3. Удалите или перевыпустите ключ
4. Обновите `.env.local` с новым ключом

## Для команды разработчиков

Каждый разработчик должен:
1. Создать свой `.env.local` файл
2. Добавить свой API ключ
3. **Никогда не коммитить** `.env.local` в git

## Проверка безопасности

Убедитесь, что `.env.local` игнорируется:
```bash
git check-ignore .env.local
# Должно вывести: .env.local
```

Если файл не игнорируется, проверьте `.gitignore`:
```bash
cat .gitignore | grep env.local
```
